  1. Que es un sistema de control de versiones?

    Un sistema de control de versiones es un modo relativamente moderno de manejar cambios, interacciones, y en si, versiones de una aplicación de software en desarrollo. El mayor provecho que se obtiene de esto proviene precisamente del desarrollo colaborativo.

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones en si, mientras que github es una herramienta que provee ese sistema de control de versiones. Es como hablar de la diferencia entre una bebida y un proveedor.
